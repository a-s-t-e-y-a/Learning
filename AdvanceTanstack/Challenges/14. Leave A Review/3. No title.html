Dominik D 14 days ago

The callbacks on useMutation will always run, while the callbacks on mutate only run IF the component is still mounted when the mutation finishes. Invalidation is something we'd always want to do. Also, only the callbacks of useMutation can return a Promise that will be internally awaited.

The callbacks on mutate are for component specific side effects, like resetting state or redirecting the user to a different page. I have some content on this in my blog
