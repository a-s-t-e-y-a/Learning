Hint #1

The useMutation hook returns an object with a mutate function that you can call to trigger the mutation. It also returns an isPending boolean that you can use to determine if the mutation is currently running, along with an isError boolean for when the mutation is in an error state. You can use these two values to disable the submit button and display a <Loader /> component while the mutation is running.

const useSubmitReview = () => {
  return useMutation({
    mutationFn: postReviewData
  });
};


const { mutate, isPending, isError } = useSubmitReview();
Hint #2

The useMutation hook has an onSuccess callback that you can use to invalidate the reviews query. This will cause the reviews query to refetch and display the new review.

const useSubmitReview = () => {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: postReviewData,
    onSuccess: () => {
      return queryClient.invalidateQueries({ queryKey: ["reviews"] });
    }
  });
};
Hint #3

You can use the additional callbacks on mutate to clear the form once the mutation has finished successfully by calling e.target.reset()

const handleSubmit = (e) => {
  e.preventDefault();
  const formData = new FormData(e.target);
  const newReview = {
    bookId,
    rating: formData.get("star-input"),
    title: formData.get("review-title"),
    text: formData.get("review-text")
  };


  mutate(newReview, {
    onSuccess: () => {
      e.target.reset();
    }
  });
};

import { useMutation, useQueryClient } from "@tanstack/react-query";
import { postReviewData } from "./utils";
import Loader from "./Loader";


const useSubmitReview = () => {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: postReviewData,
    onSuccess: () => {
      return queryClient.invalidateQueries({ queryKey: ["reviews"] });
    }
  });
};
